SILO CFW — إضافة رول تلقائي عند قبول أو رفض طلب التفعيل

هذه النسخة مبنية فوق نسخة تسجيل الدخول بحساب Discord.

السلوك:
- عند الضغط على "قبول الطلب":
  1) يضاف رول القبول.
  2) ينحذف رول الرفض إذا كان موجوداً.
  3) تتحول حالة الطلب إلى مقبول.

- عند الضغط على "رفض الطلب":
  1) يضاف رول الرفض.
  2) ينحذف رول القبول إذا كان موجوداً.
  3) تتحول حالة الطلب إلى مرفوض.

- إذا فشلت إضافة الرول، يبقى الطلب قيد المراجعة ولا تُقفل الأزرار.

==================================================
1) أنشئ الرولين داخل Discord
==================================================

مثال:
- مفعل
- مرفوض

فعّل Developer Mode من:
User Settings > Advanced > Developer Mode

بعدها:
- كلك يمين على السيرفر > Copy Server ID
- كلك يمين على رول "مفعل" > Copy Role ID
- كلك يمين على رول "مرفوض" > Copy Role ID

==================================================
2) صلاحيات وترتيب رتبة البوت
==================================================

Discord Server Settings > Roles

1. افتح رتبة بوت SILO Whitelist.
2. فعّل صلاحية:
   Manage Roles
3. اسحب رتبة البوت إلى فوق رول "مفعل" وفوق رول "مرفوض".

الترتيب الصحيح مثال:

SILO Whitelist Bot
مفعل
مرفوض
@everyone

إذا رتبة البوت تحت الرولين، لن يستطيع إضافتهما حتى لو عنده Manage Roles.

==================================================
3) متغيرات Netlify الجديدة
==================================================

أضف داخل:
Netlify > Project configuration > Environment variables

DISCORD_GUILD_ID=آيدي السيرفر
DISCORD_ACCEPTED_ROLE_ID=آيدي رول القبول
DISCORD_REJECTED_ROLE_ID=آيدي رول الرفض

اكتب Key وحده وخانة Value وحدها، لا تكتب علامة = داخل خانة Key.

مثال:
Key: DISCORD_GUILD_ID
Value: 123456789012345678

خلي القيم متاحة للـ Functions أو All scopes حسب خطتك.

لا تحذف المتغيرات السابقة:
DISCORD_BOT_TOKEN
DISCORD_PUBLIC_KEY
DISCORD_CHANNEL_ID
DISCORD_REVIEWER_ROLE_ID
DISCORD_CLIENT_ID
DISCORD_CLIENT_SECRET
AUTH_SESSION_SECRET

==================================================
4) تحديث GitHub
==================================================

فك ضغط الملف، وارفع جميع محتوياته إلى نفس مستودع:
silo-cfw-website

وافق على استبدال الملفات القديمة ثم Commit changes.

Netlify سيبدأ Deploy تلقائياً.

بعد إضافة متغيرات Netlify:
Deploys > Trigger deploy > Deploy project

انتظر حتى تصبح الحالة Published.

==================================================
5) الاختبار
==================================================

1. ادخل بحساب Discord داخل الموقع.
2. قدم طلب تفعيل.
3. افتح روم طلبات التفعيل.
4. اضغط قبول.
5. تأكد أن رول القبول انضاف للمتقدم.
6. جرّب طلب آخر واضغط رفض.
7. تأكد أن رول الرفض انضاف.

إذا ظهر:
"المتقدم مو موجود داخل سيرفر الديسكورد"
فلازم يدخل الشخص إلى السيرفر قبل التقديم.

إذا ظهر:
"البوت ما عنده صلاحية Manage Roles..."
راجع صلاحية Manage Roles وترتيب رتبة البوت.

==================================================
ملاحظة أمنية
==================================================

لا ترسل Bot Token أو Client Secret لأي شخص.
لا تضعها داخل GitHub أو ملفات الموقع.
تبقى داخل Environment variables في Netlify فقط.
